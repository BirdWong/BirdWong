graph LR
    subgraph 服务器1
    A(myid1:1)---B(ip:172.17.0.2)
    end
    subgraph 服务器2
    C(myid1:2)---D(ip:172.17.0.3)
    end
    subgraph 服务器3
    F(myid1:3)---E(ip:172.17.0.4)
    end
  1. 服务器1启动， 发现无leader，发起选举， 只有自己，投票自己，少于半数，选举失败
  2. 服务器2启动， 发现无leader，发起选举，服务器2自己投自己，服务器1自己投自己，选举失败，均未过半
  3. 服务器1服务器2发现无leader， 发起选举， 服务器2 id > 服务器1 id， 服务器1选举服务器2， 服务器2选举服务器2， 超过半数。 选举成功
  4. 服务器3启动， 已经有leader， 不进行选举， 自动转为follower

节点

节点类型

通过create中的可选参数[-e]指定

  • 短暂节点：断开链接后自动删除，可以用于服务器注册，一旦服务器挂掉，zookeeper就会自动删除
  • 持久节点：断开不删除，除非执行删除命令

-e说明是短暂节点， 如果不申明，说明是持久节点

节点形式

  • 短暂节点
  • 短暂节点带序号
  • 持久节点（默认）
  • 持戒节点带序号

带序号说明在同一个节点下建立新的节点，在create命令后带-s新的节点名后面会自动增加一个递增序列id
